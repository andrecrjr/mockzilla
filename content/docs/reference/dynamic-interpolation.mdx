# Dynamic Data Syntax

Mockzilla lets you build cohesive data where one field references another using a powerful interpolation system.

## 1. Type Preservation vs. String Interpolation

Mockzilla's engine is intelligent about data types. Knowing *how* to write your templates is the best way to ensure your frontend receives exactly what it expects.

### Strict Type Preservation (Best for Booleans & Numbers)
If a template occupies the **entire string**, Mockzilla preserves its original type.
- `"isActive": "{{state.is_active}}"` → Returns a real `boolean`.
- `"count": "{{db.items.length}}"` → Returns a real `number`.
- `"data": "{{state.user_object}}"` → Returns a real `object`.

### String Interpolation (Best for Messages)
If you add any text around the template, it is forcibly converted to a string.
- `"msg": "User status is {{state.status}}"` → Returns `"User status is active"`.

---

## 2. Request Context Interpolation (`{$.path}`)
You can reuse anything sent by the frontend in your response. This works in Static Responses, Wildcard Variants, and Dynamic Schemas.

| Syntax | Description | Example |
|--------|-------------|---------|
| `{{input.query.page}}` | Injects a URL query parameter. | `?page=2` → `{"page": 2}` |
| `{{input.body.id}}` | Injects a value from the request JSON. | `{"id": "abc"}` → `{"ref": "abc"}` |
| `{{input.params.id}}` | Injects a URL path segment. | `/users/123` → `{"userId": 123}` |

---

## 3. Resilient Arithmetic (The "Safe Math" Way)
You can perform math directly in templates. Mockzilla is built to be resilient—it will never crash your workflow if data is missing.

- **Success**: `{{state.count + 1}}` → `11`.
- **Graceful Fallback**: If `state.score` is missing, `{{state.score + 10}}` simply returns the raw string `{{state.score + 10}}` rather than erroring, allowing you to debug your state easily.

---

## 4. Supercharged Faker
Want to generate incredibly realistic data? Use the `{{faker.path}}` syntax anywhere.

### Best Ways to Use Faker
- **Simple Calls**: `{{faker.internet.email}}`
- **With Arguments**: Use JSON to constrain results.
  `{{faker.number.int({"min": 10, "max": 100})}}`
- **Dynamic Arguments**: You can even interpolate *into* a Faker call!
  `{{faker.string.alphanumeric(input.query.len)}}`

---

## 5. Power Moves: Advanced Relational Lookups
Mockzilla supports **advanced predicates** for array lookups, which is the best way to simulate database queries.

| Syntax | Description |
|--------|-------------|
| `db.users[id=123].name` | Finds the user where `id` is `123`. |
| `db.sessions[token=input.headers.auth]` | **Dynamic Lookup**: Matches a session by an incoming header. |

> **Pro Tip**: You can chain these lookups! `db.orders[id=1].items[0].price` finds the first item of order #1 and returns its price.
