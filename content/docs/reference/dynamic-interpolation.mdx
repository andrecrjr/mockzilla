# Dynamic Data Syntax

Mockzilla lets you build cohesive data where one field references another using a powerful interpolation system.

## 1. Internal Referencing (`{$.path}`)
You can build objects where one property depends on the value of another property in the same object.

| Syntax | What it Does | Example |
|--------|--------------|---------|
| `{$.id}` | Injects the value of the `id` field. | `{ "id": "123", "orderRef": "ORD-{$.id}" }` |
| `{{$.name}}` | Injects a value into a string. | `"Hello {{$.name}}!"` |

### Complex Example:
```json
{
  "firstName": { "faker": "person.firstName" },
  "lastName": { "faker": "person.lastName" },
  "email": { "pattern": "{{$.firstName}}.{{$.lastName}}@example.com" }
}
```

---

## 2. Request Context Interpolation (`{$.query.field}`)
Mockzilla provides the request context to the mocking engine, allowing you to reuse what the frontend sent in **ANY** response (Static, Variant, or Dynamic Schema).

| Syntax | What it Does | Example |
|--------|--------------|---------|
| `{$.query.page}` | Injects a URL query parameter. | `?page=2` → `{ "meta": { "page": "{$.query.page}" } }` |
| `{$.query.limit}`| Injects a limit parameter. | `?limit=50` → `{ "limit": "{$.query.limit}" }` |

### Where to use it:
- **Static Response Tab**: Echo back query params in your hand-written JSON.
- **Dynamic Schema**: Reference query params in `const` or `pattern` fields.
- **Wildcard Variants**: Personalize variant responses based on the request.

### Practical Use:
If your mock response is:
`{"message": "Your value is {$.query.val}"}`

Calling `?val=banana` will return:
`{"message": "Your value is banana"}`

## 3. Supercharged Faker

Mockzilla doesn't just support standard Faker—it gives you two ways to generate incredibly realistic data.

### Method A: JSON Schema Formats (The Easy Way)
We've added high-level **semantic formats** to JSON Schema. Just set the `format` and Mockzilla does the rest!

| Format | Output Example |
|--------|----------------|
| `password` | `zP8_x9L1q` |
| `phone` | `(555) 123-4567` |
| `country` | `United Kingdom` |
| `currency`| `USD` |
| `color` | `sky blue` |

**Example:**
```json
{
  "user_password": { "type": "string", "format": "password" },
  "support_phone": { "type": "string", "format": "phone" }
}
```

### Method B: Direct Faker Calls (The Flexible Way)
Want to use Faker anywhere—even in static JSON or custom strings? Use the `{{faker.path}}` syntax.

| Syntax | Result |
|--------|--------|
| `{{faker.internet.email}}` | `john.doe@gmail.com` |
| `Hello, {{faker.person.firstName}}!` | `Hello, Alice!` |
| `{{faker.number.int({"min": 10, "max": 100})}}` | `42` (Deterministic range) |
| `{{faker.string.alphanumeric(10)}}` | `a1b2c3d4e5` (Fixed length) |

> **Pro Tip**: You can pass **JSON objects** or simple values as arguments to any Faker method! This works even in **Static Responses**.

## 4. Power Moves: Advanced Examples

Combine internal referencing, request context, and direct Faker calls to create "Production-Ready" mocks.

### A. The "Smart" E-commerce Product
Generate a product that respects its ID format and has a realistic price range.
```json
{
  "id": "PRD-{{faker.string.alphanumeric(8)}}",
  "name": "{{faker.commerce.productName}}",
  "price": {{faker.commerce.price({"min": 100, "max": 500, "symbol": "$"})}},
  "description": "This {{faker.commerce.productAdjective}} item is a must-have!",
  "stock": {{faker.number.int({"min": 0, "max": 50})}}
}
```

### B. Personalized API Greeting
Echo a user's name from a query parameter and generate a unique session token.
```json
{
  "message": "Welcome back, {{input.query.name}}!",
  "session": {
    "id": "{{faker.string.uuid}}",
    "token": "{{faker.string.alphanumeric(32)}}",
    "expires": "{{faker.date.future({"years": 1})}}"
  }
}
```

### C. Realistic Transaction List
Use arguments to constrain data types and ensure they match your database expectations.
```json
{
  "transactions": [
    {
      "id": "{{faker.string.nanoid(10)}}",
      "amount": "{{faker.finance.amount({"min": 5, "max": 1000})}}",
      "currency": "{{faker.finance.currencyCode}}",
      "status": "{{faker.helpers.arrayElement(["pending", "completed", "failed"])}}",
      "card": "**** **** **** {{faker.finance.creditCardCVV}}"
    }
  ]
}
```

### D. Deeply Nested Profile
Construct complex objects that match your exact frontend requirements.
```json
{
  "user": {
    "profile": {
      "username": "{{faker.internet.userName}}",
      "avatar": "{{faker.image.avatar}}",
      "preferences": {
        "theme": "{{faker.helpers.arrayElement(["dark", "light", "system"])}}",
        "notifications": {{faker.datatype.boolean}}
      }
    },
    "history": {
      "lastLogin": "{{faker.date.recent}}",
      "location": {
        "city": "{{faker.location.city}}",
        "zip": "{{faker.location.zipCode("#####")}}"
      }
    }
  }
}
```

## 5. Formatting Rules
When using **Response Templates** (not JSON Schema), you can still interpolate request data:

| Source | Syntax | Description |
|--------|--------|-------------|
| **Path** | `{{input.params.id}}` | Value from a path parameter. |
| **Body** | `{{input.body.name}}` | Value from the JSON request body. |
| **Headers**| `{{input.headers.auth}}` | Value from an HTTP header. |
| **Query** | `{{input.query.limit}}` | Value from the URL query string. |

> **Pro Tip**: Use these in the **Static Response** tab to echo back what the user sent you!
