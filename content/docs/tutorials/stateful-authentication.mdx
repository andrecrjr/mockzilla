# Tutorial: Building a Stateful Auth Flow

In this tutorial, you will build a realistic authentication simulation. Unlike static mocks, Mockzilla Workflows can **remember** if a user has logged in, allowing you to test protected routes and session expiration logic in your frontend.

## What You'll Learn
- How to store data in the **Scenario State**.
- How to use **Conditions** to protect routes.
- How to use **Faker.js** to generate dynamic tokens.
- How to personalize responses based on state.

---

## Step 1: Initialize the Scenario
1.  Navigate to the **Workflows** tab in the Mockzilla Dashboard.
2.  Click **Create Scenario** and name it `auth-demo`.
3.  Note your Execution URL: `http://localhost:36666/api/workflow/exec/auth-demo/`

---

## Step 2: Create the Login Endpoint
We need an endpoint that accepts credentials and "sets" a session token in our state.

1.  Click **Add Transition**.
2.  **Name**: `User Login`
3.  **Path**: `/login`
4.  **Method**: `POST`
5.  **Effects**: Add a `state.set` effect.
    - **Key**: `token`
    - **Value**: `{{faker.string.alphanumeric(32)}}`
    - *Also add another `state.set` for the username:*
    - **Key**: `currentUser`
    - **Value**: `{{input.body.username}}`
6.  **Response**:
    ```json
    {
      "success": true,
      "token": "{{state.token}}",
      "user": {
        "name": "{{state.currentUser}}",
        "lastLogin": "{{faker.date.recent()}}"
      }
    }
    ```

---

## Step 3: Protect the Profile Route
Now, let's create a `/profile` endpoint that **only** responds if the correct token is provided in the `Authorization` header.

1.  Click **Add Transition**.
2.  **Name**: `Get User Profile`
3.  **Path**: `/profile`
4.  **Method**: `GET`
5.  **Condition**: This is the secret sauce.
    - **Selector**: `input.headers.authorization`
    - **Operator**: `==`
    - **Value**: `Bearer {{state.token}}`
6.  **Response**:
    ```json
    {
      "id": "{{faker.string.uuid()}}",
      "username": "{{state.currentUser}}",
      "email": "{{faker.internet.email()}}",
      "roles": ["user", "beta-tester"]
    }
    ```

---

## Step 4: Handle "Unauthorized" Errors
If the token doesn't match, we want a helpful error instead of a generic 404.

1.  Click **Add Transition**.
2.  **Name**: `Unauthorized Access`
3.  **Path**: `/profile`
4.  **Method**: `GET`
5.  **Condition**: (Leave empty - it will catch anything the previous transition missed).
6.  **Response**:
    - **Status Code**: `401`
    - **Body**:
      ```json
      { "error": "Unauthorized", "message": "Valid session token required" }
      ```

---

## Step 5: Failed Login Lockout (Advanced)
Let's add logic to "lock" an account after 3 failed attempts. This uses **Arithmetic** to increment a counter and **Conditions** to branch logic.

1.  Click **Add Transition**.
2.  **Name**: `Login Failed`
3.  **Path**: `/login`
4.  **Method**: `POST`
5.  **Condition**: `input.body.password != "password123"`
6.  **Effects**: `state.set`
    - **Key**: `login_attempts`
    - **Value**: `{{state.login_attempts + 1}}`
7.  **Response**:
    ```json
    { "error": "Invalid credentials", "attempts": "{{state.login_attempts}}" }
    ```

8.  Click **Add Transition** (and move it to the top of the list).
9.  **Name**: `Account Locked`
10. **Path**: `/login`
11. **Method**: `POST`
12. **Condition**: `state.login_attempts >= 3`
13. **Response**:
    - **Status Code**: `423` (Locked)
    - **Body**: `{ "error": "Account locked due to 3+ failed attempts." }`

---

## Step 6: Test the Flow

### 1. Attempt unauthorized access
```bash
curl -i http://localhost:36666/api/workflow/exec/auth-demo/profile
```
**Result**: `401 Unauthorized`.

### 2. Login to generate a session
```bash
curl -X POST http://localhost:36666/api/workflow/exec/auth-demo/login \
  -H "Content-Type: application/json" \
  -d '{"username": "Alice", "password": "password123"}'
```
**Result**: You'll get a JSON response with a random token. **Copy that token.**

### 3. Access profile with your token
Replace `YOUR_TOKEN` with the one from the previous step:
```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  http://localhost:36666/api/workflow/exec/auth-demo/profile
```
**Result**: `200 OK` with Alice's profile data!
