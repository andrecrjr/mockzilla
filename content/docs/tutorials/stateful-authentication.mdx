# Tutorial: Building a Stateful Auth Flow

In this tutorial, you will build a realistic authentication simulation. Unlike static mocks, Mockzilla Workflows can **remember** if a user has logged in, allowing you to test protected routes and session expiration logic in your frontend.

We need an endpoint that validates credentials and "sets" a session token in our state.

      <div className="grid grid-cols-2 gap-2">
        <div className="p-2 bg-muted rounded text-xs">`token` → `&#123;&#123;faker.string.alphanumeric 32&#125;&#125;`</div>
        <div className="p-2 bg-muted rounded text-xs">`currentUser` → `&#123;&#123;input.body.username&#125;&#125;`</div>
      </div>
    </div>
  </CardContent>
</Card>

<Alert className="mt-4 border-dashed">
  <Zap className="h-4 w-4" />
  <AlertTitle>Pro-Tip: Faker in Effects</AlertTitle>
  <AlertDescription>
    Using `&#123;&#123;faker&#125;&#125;` inside a `state.set` effect ensures that the token is generated once during login and remains persistent until the next login.
  </AlertDescription>
</Alert>
