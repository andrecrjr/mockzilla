# Dynamic Data Generation

Mockzilla's most powerful feature is its ability to turn a simple **JSON Schema** into a rich, realistic API response. Stop handwriting massive arrays of static objects — let Mockzilla generate them for you!

## How it Works

In the **Response Config** tab of any mock, toggle the **From Schema** switch.

1. **Write your schema:** Use standard [JSON Schema](https://json-schema.org/) to define the structure.
2. **Add Faker commands:** Use the `faker` keyword to get realistic names, emails, and dates.
3. **Get fresh data:** Every time you hit the mock's URL, a new, randomized response is generated.

## 🚀 Copy-Paste Recipes

### 1. Simple User List
Get an array of 5 to 10 users with avatars.

```json
{
  "type": "array",
  "minItems": 5,
  "maxItems": 10,
  "items": {
    "type": "object",
    "properties": {
      "id": { "type": "string", "format": "uuid" },
      "name": { "type": "string", "faker": "person.fullName" },
      "avatar": { "type": "string", "faker": "image.avatar" }
    },
    "required": ["id", "name"]
  }
}
```

### 2. Product Catalog
Generate realistic prices and descriptions.

```json
{
  "type": "array",
  "items": {
    "type": "object",
    "properties": {
      "id": { "type": "string", "faker": "string.uuid" },
      "title": { "type": "string", "faker": "commerce.productName" },
      "price": { "type": "string", "faker": "commerce.price" },
      "tags": { "type": "array", "faker": "commerce.productAdjective" }
    }
  }
}
```

## Master Interpolation Syntax

You can build cohesive data where one field references another using the `{$.path}` syntax.

| Syntax | What it Does | Example |
|--------|--------------|---------|
| `{$.id}` | Reuses the value of the `id` field. | `{ "id": "123", "orderRef": "ORD-{$.id}" }` |
| `{{$.name}}` | Injects a value into a string. | `"Hello {{$.name}}!"` |
| `faker: "person.name"` | Calls a specific [Faker.js](https://fakerjs.dev/) method. | Generates "John Smith" |

### Referring to Other Fields
You can even build complex strings like:
```json
{
  "firstName": { "faker": "person.firstName" },
  "lastName": { "faker": "person.lastName" },
  "email": { "pattern": "{{$.firstName}}.{{$.lastName}}@example.com" }
}
```

> **Pro Tip:** Dynamic responses are fresh on every request. This is perfect for testing how your UI handles long lists, short lists, or varying data types.
