# Overview

Mockzilla is a high-fidelity API mocking tool that lets you simulate complex backend responses without writing a single line of server code. Whether you're building a new feature or debugging a tricky user flow, Mockzilla provides the tools you need to stay in the zone.

<div id="overview" />

## Why Mockzilla?

Traditional mocks are static and boring. Mockzilla makes them **alive**:

| Feature | User Value |
|---------|------------|
| **Dynamic Data** | Stop handwriting massive JSON files. Generate 1,000 realistic users or orders with one click. |
| **Routing** | Match requests by exact path, wildcards, or even specific query parameters. |
| **Workflows** | Simulate stateful flows (like a shopping cart or login) using a built-in mini-database. |
| **AI Integration** | Let your AI agent (Cursor, Claude) create and manage mocks for you via MCP. |
| **Extension** | Intercept real browser requests and sync them to your dashboard instantly. |

## Installation

Choose the tool that fits your workflow.

### 1. HTTP Server (Docker)
The standalone server is perfect for teams, local development, or cloud hosting (VPS). It provides a dashboard to manage all your mocks.

```bash
docker run -d -p 36666:36666 -v mockzilla-data:/app/data --name mockzilla andrecrjr/mockzilla:latest
```

**Note for VPS users:** Mockzilla runs on port **36666** by default. Ensure this port is open in your firewall/security groups to access the dashboard and mocks remotely.

Once running, access your instance at:
`http://<your-ip-or-domain>:36666`

### 2. Chrome Extension
For real-time browser interception, install the official extension. It allows you to mock requests directly in your browser without changing your code.

- **[Install from Chrome Web Store](https://chromewebstore.google.com/detail/gpbfcijfelgnaabjodgifeopbmnbjpnk/preview)**

## Quick Start (The UI Way)

Getting started with Mockzilla is designed to be intuitive and visual.

### 1. Create a Folder
Folders are how you organize your projects. Click **New Folder** on the dashboard to give your workspace a name.

### 2. Add a Mock
Inside your folder, click **Create Mock**. You can define the endpoint path (e.g., `/users`), the HTTP method, and the status code you want to return.

### 3. Configure the Response
In the **Response Config** tab, you can paste static JSON or switch to **Dynamic Response** to use powerful templates.

### 4. Test it Out
Every folder has a unique "Serve URL". Once your mock is saved, any request to that URL will return your configured response!
