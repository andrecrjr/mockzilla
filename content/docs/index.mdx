# Welcome to Mockzilla

Mockzilla is a high-fidelity API mocking platform designed to bridge the gap between frontend development and backend reality. From simple static responses to complex stateful simulations, we've got you covered.

## Why Mockzilla?

Mockzilla is designed for the modern development ecosystem, where speed, product validation, and cost-efficiency are paramount.

### Core Strengths (The Pros)
- **🚀 Product-First approach**: Deliver as fast as possible by mocking first. Establish Mockzilla as the **Ground Truth**—design your API contract and validate your UI before writing a single line of backend code.
- **🤖 AI-Native Power-Ups**: Built-in MCP support allows AI agents (like **Claude Opus**, **GPT**, or **Gemini**) to autonomously build, debug, and maintain your mock infrastructure while minimizing token burn.
- **🎭 Stateful "Living" Mocks**: Unlike static JSON servers, Mockzilla supports complex stateful scenarios with an in-memory Mini-DB and session state—perfect for BFFs and Microservice prototypes.
- **🎲 High-Fidelity Dynamic Data**: Use JSON Schema + Faker.js to generate fresh, realistic data that adheres strictly to your API contracts.
- **🔌 Browser-to-Dashboard Sync**: Capture real-world production traffic via our Chrome Extension and convert it into high-fidelity mocks with one click.
- **🧠 Smart Hybrid Engine**: Automatically switches between type-preserving JSON interpolation and powerful Handlebars logic for maximum flexibility.
- **🐳 DevOps Friendly**: Standalone Docker support and a robust Makefile-driven CLI for seamless integration into local and CI/CD environments.
- **💸 Token Burn Mitigation**: Drastically reduce AI SaaS costs by mocking expensive LLM responses during UI/UX development—stop wasting your token budget on layout refreshes.
- **🔍 Forensic Debugging**: Built-in observability tools allow you to trace exactly why a specific mock matched (or didn't) and how state mutated during a request.

### Points of Consideration (The Cons)
- **📈 Learning Curve**: Mastering advanced Handlebars interpolation and stateful transition logic requires some initial time investment.
- **🔄 Mock Drift**: Like any mocking tool, you must maintain discipline to keep mocks in sync with evolving production backend contracts.
- **🧩 Logic Complexity**: While powerful, building highly complex stateful simulations can become difficult to manage without proper modularization.

## Role-Based Benefits

### For Product Teams
Deliver a functional, interactive prototype in days, not months.
- **Pros:** Validate product-market fit against a "Living Mock" without waiting for a full MVP backend; perfect for rapid iteration.
- **Cons:** Requires clear specification of the product's API contract.

### For Frontend Developers
Parallelize development by working against high-fidelity mocks before the backend exists.
- **Pros:** Mock complex aggregations (BFFs) and downstream microservices; use the Chrome extension to capture production traffic.
- **Cons:** Requires discipline to maintain mock accuracy as UI requirements evolve.

### For AI Engineers
Native MCP support allows AI assistants to autonomously create and manage mocks within your workspace.
- **Pros:** Zero-latency testing for agent tool-calling; AI can build complex stateful scenarios in seconds.
- **Cons:** Requires initial MCP server configuration.

### For QA & Automation
Reproduce edge cases that are hard to trigger in real staging environments.
- **Pros:** Instantly simulate 500 errors, timeouts, or specific "dirty" data; test stateful flows without backend seeding.
- **Cons:** Not a replacement for full, end-to-end integration testing.
